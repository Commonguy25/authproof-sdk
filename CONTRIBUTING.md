# Contributing to Authproof

Thanks for your interest in contributing.

## How to contribute

- Open an issue to report bugs or suggest features
- Submit a pull request with improvements
- Cryptographic attack vectors especially welcome — if you find a flaw in the protocol design open an issue immediately

## Development

npm install
npm test

All 573 tests must pass before submitting a pull request.

## Protocol contributions

If you want to contribute to the protocol specification rather than the implementation open an issue with the tag [spec] and describe the proposed change and your reasoning.

## Contact

Open an issue or reach out via authproof.dev
